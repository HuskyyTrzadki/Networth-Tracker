# Page Override - Transactions

Use `MASTER.md` as baseline. This file only lists transactions-specific rules.

## Table Tone
- Table is the product here: keep separators subtle, rows stable, and typography consistent.
- Default row state should look calm; hover/selected should be visible but not loud.

## Badge Usage
- Transaction type/status badges default to neutral stamp style.
- Use strong semantic color only for high-signal states.

## Scanability
- Keep column labels clear and concise in Polish.
- Maintain mono + tabular values for price/quantity/value fields.
- Avoid UI noise around filters/sorting controls; clarity over decoration.
